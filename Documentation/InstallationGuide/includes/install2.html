Installing: Step 2 - Installation (continued)


Step 3) The next step is to create a new MySQL database and import the included main.sql file in the /config/sql/ folder, into your database.  You should create a new database (we don't recommend using an existing one), and set up a username & password for it which you can remember. You can do this by accessing the "MySQL" options from your Host’s Control Panel or within Plesk/Cpanel. If you need help with this, please contact your web host, or view our Videos in the Support Center.  To reiterate, in the downloaded files you have, go to the /config/sql/ folder and import the main.sql file into your new MySQL database. You may wish to do this via the unix shell prompt, or phpMyAdmin (recommended) if you are familiar with that. This will import all necessary data & tables into your database.




Step 4) Now you need to edit the configuration (config.php) file. This is a core file where all of the major settings are.  On your server, in the /config/ folder you will see a file called 'config.default.php'. Rename this to 'config.php', and fill out all the details by following the instructions inside. You will need to make sure to enter a domain name, email address and a few other deetails. You will also need to include your MySQL database name, username, password, and hostname, from the step above, in the config.php file. In most cases, the hostname will be “localhost”. You can obtain the rest of the information from your web hosting provider if unsure. Note that the 'prefix' field should nearly always be left blank.

Then, go through and fill out your domain name where you are using the software. For example the 'ref_url', 'nml_url' and 'serverName' fields need to be filled out.

 

Leave the rest of the settings as they are for the time being and save the file with the changes made.





Step 5) The next step is to set some permissions on folders. We do this via chmod (set permissions to) 777 on the following directories:

           /tmp/

           /webroot/img/
         

  File/Folder Name Item CHMOD Value Recursively (Files/Folders Within) CHMOD Needed?
/tmp/ 0777YES YES
/webroot/img/ 0777 YES YES


NB: Make sure ALL folders inside these are also set recursively to 777 (read/write/all).  For example this folder must also be set with the same permissions:

           /tmp/cache/

 

All other folders should be set to the default server setting, which is normally 755. Do not set any other folders to 777.

 


Tip: if you are unsure how to CHMOD a folder or file, please see here: http://www.phpjunkyard.com/ftp-chmod-tutorial.php, or contact phpPennyAuction support.
